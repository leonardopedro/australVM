OBJECTIVE: Integrate CpsGen into main Compiler.ml pipeline

COMPONENT CREATED:

lib/Compiler_cps.ml (6,000+ bytes)
  └─ Bridges MAST (Monomorphic AST) → CpsGen
  
  KEY FUNCTIONS:
  • convert_expr : Mt.mexpr → CpsGen.cps_expr
    - Handles: Int constants, booleans, variables, function calls
    - Converts: MIntConstant → IntLit, MApp → App, etc.
    
  • convert_stmt : Mt.mstmt → CpsGen.cps_stmt  
    - Handles: Let, If, Return, Assign, Block
    - Skips: Borrow, While, For, Destructure (Phase 7)
    
  • build_cps_function : Mt.mdecl → CpsGen.function_def option
    - Filters: MFunction and MConstant declarations
    - Converts: Parameter lists, return types, body
    
  • compile_module_cps : Mt.mono_module → CpsGen.function_def list
    - Entry point: Takes module, returns list of CPS functions
    
  • write_cps_binary : Binary output for Rust bridge

INTEGRATION POINT IN Compiler.ml:

  Original flow (line 169):
    let mono = monomorphize env typed in
    let unit = gen_module env mono in      ← C codegen
    render_unit unit                       → C string

  CPS-enhanced flow:
    let mono = monomorphize env typed in
    if !use_cps_jit then
      let funcs = Compiler_cps.compile_module_cps mono in
      let binary = CpsGen.serialize_functions funcs in
      let fn_ptr = CamlCompiler_rust_bridge.compile_mast binary in
      store_pointer fn_ptr                   ← JIT path (O(1) stack)
    else
      gen_module env mono                    ← C codegen (fallback)
